6. Load Management: Implemented ethical scraping practices to manage the load on competitor apps' servers, including rate limiting, caching, randomized intervals, user-agent rotation, and IP rotation. This ensured minimal impact on the target servers while maintaining efficient data collection.

Project Title: SFTP Kafka Connector
Project Overview: I contributed to significantly advancing our application's data integration and security capabilities. A central facet of this project was implementing an efficient, event-driven system using Apache Kafka to revolutionize how data was handled, processed, and distributed within the application's ecosystem.
Key Features:
1. SFTP to Kafka Integration: Successfully designed and executed the integration of a secure SFTP location with our Kafka cluster. The SFTP location served as a data producer, where one of our clients regularly updated datasets. This data was then seamlessly ingested into Kafka, providing a consistent and reliable data flow.
2. Data Transformation and Management: Implemented sophisticated logic to transform raw data from Kafka's sink, MongoDB, into a structured format aligning with MetricStream's specific data models and terminologies. This crucial step ensured data uniformity and compatibility with our application's standards.
3. Custom API Development: Engineered custom APIs which allowed other MetricStream teams to access the processed data. This development facilitated the accurate representation of data within the broader MetricStream platform, enabling various downstream applications and services to utilize this data effectively.
4. Security with OAuth2: Enforced robust security measures by integrating OAuth2 protocols at both ends of the data flow — securing data access from the SFTP producer and ensuring secure data consumption by the APIs. This dual implementation fortified the security posture of our data streams, safeguarding sensitive information throughout the process.
